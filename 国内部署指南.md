# 国内访问部署指南

## 🇨🇳 国内访问解决方案

### 问题说明
由于网络环境限制，直接访问 GitHub Pages、Vercel 等国外托管服务可能会遇到访问困难。

---

## 🚀 推荐的国内部署方案

### 方案一：Gitee Pages（推荐）

**优点：**
- ✅ 国内访问速度快
- ✅ 完全免费
- ✅ 部署简单
- ✅ 支持自定义域名

**部署步骤：**

1. **注册 Gitee 账号**
   - 访问：https://gitee.com/
   - 注册并登录

2. **创建仓库**
   - 点击右上角 "+" → "新建仓库"
   - 仓库名：`permit2-index`
   - 选择"公开"
   - 点击"创建"

3. **导入 GitHub 代码**
   - 在仓库页面点击"从 GitHub / GitLab 导入仓库"
   - 输入 GitHub 仓库地址：`https://github.com/yeomen1/permit2-index.git`
   - 点击"导入"

4. **启用 Gitee Pages**
   - 进入仓库页面
   - 点击"服务" → "Gitee Pages"
   - 选择分支：`main`
   - 部署目录：`/` (根目录)
   - 点击"启动"
   - 等待部署完成（通常 1-2 分钟）

5. **访问地址**
   - 部署成功后会显示访问地址，例如：
   - `https://your-username.gitee.io/permit2-index/`

### 方案二：Coding Pages（腾讯云）

**优点：**
- ✅ 腾讯云托管，稳定可靠
- ✅ 国内访问速度快
- ✅ 支持 CI/CD

**部署步骤：**

1. **注册 Coding 账号**
   - 访问：https://coding.net/
   - 使用微信或邮箱注册

2. **创建项目**
   - 点击"新建项目"
   - 选择"DevOps 项目"
   - 项目名称：`permit2-index`

3. **导入代码**
   - 选择"从 Git 仓库导入"
   - 输入 GitHub 仓库地址

4. **启用静态网站服务**
   - 进入项目
   - 点击左侧"持续部署" → "静态网站"
   - 点击"新建静态网站"
   - 选择仓库和分支
   - 配置完成后点击"保存"

5. **访问地址**
   - 部署成功后会提供访问地址

### 方案三：本地服务器（开发测试）

**使用 Python 快速启动：**

```bash
# Python 3
cd c:\Users\YE\Desktop\permit2-index
python -m http.server 8080

# 访问地址：http://localhost:8080/
```

**使用 Node.js：**

```bash
# 安装 http-server
npm install -g http-server

# 启动服务器
cd c:\Users\YE\Desktop\permit2-index
http-server -p 8080

# 访问地址：http://localhost:8080/
```

**使用 PHP：**

```bash
# PHP 内置服务器
cd c:\Users\YE\Desktop\permit2-index
php -S localhost:8080

# 访问地址：http://localhost:8080/
```

### 方案四：使用代理访问（临时方案）

**使用 Vercel + 代理：**

1. **开启系统代理**
   - 使用 VPN 或代理软件
   - 确保代理设置为系统代理模式

2. **访问 GitHub Pages**
   - 直接访问：`https://yeomen1.github.io/permit2-index/`
   - 或访问：`https://labtoken.vercel.app/`

3. **注意事项**
   - 需要保持代理开启
   - 访问速度可能较慢
   - 不适合长期使用

## 📱 移动端测试方案

### 国内移动端测试

**方案一：使用 Gitee Pages**
1. 在手机浏览器中打开 Gitee Pages 地址
2. 测试钱包连接功能

**方案二：局域网访问**
1. 确保手机和电脑在同一 WiFi 网络
2. 启动本地服务器
3. 获取电脑 IP 地址（运行 `ipconfig` 查看）
4. 在手机浏览器访问：`http://电脑IP:8080/`

**方案三：使用内网穿透工具**
- **ngrok**: `ngrok http 8080`
- **frp**: 配置内网穿透服务
- **花生壳**: 使用花生壳内网穿透

## 🔧 钱包深度链接问题修复

### 问题描述
钱包应用打开了，但没有在钱包内打开网页

### 解决方案
已修复深度链接编码问题：
- ✅ 使用 `encodeURIComponent()` 正确编码 URL
- ✅ 支持多种钱包的深度链接格式
- ✅ 自动检测设备类型（iOS/Android）

### 测试建议
1. 确保已安装钱包应用
2. 在手机浏览器中打开网页
3. 点击"连接钱包"
4. 选择钱包应用
5. 钱包应该会打开并显示您的网页

## 🆘 常见问题

### Q1: Gitee Pages 部署失败？
**A:** 检查以下几点：
- 确保仓库是公开的
- 检查分支名称是否为 `main`
- 等待 2-3 分钟后刷新页面
- 查看部署日志了解错误详情

### Q2: 移动端无法唤起钱包？
**A:** 检查以下几点：
- 确保已安装对应的钱包应用
- 检查 URL 是否正确编码
- 尝试使用不同的浏览器
- 清除浏览器缓存后重试

### Q3: 国内访问速度慢？
**A:** 建议使用：
- Gitee Pages（推荐）
- Coding Pages
- 国内云服务器（阿里云、腾讯云等）

### Q4: 如何自定义域名？
**A:** 在各平台设置中：
- Gitee Pages：仓库设置 → Pages → 自定义域名
- Coding Pages：静态网站设置 → 域名管理
- GitHub Pages：仓库设置 → Pages → Custom domain
